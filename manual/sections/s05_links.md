> **7. Wiki 링크 1차 강화**

링크가 없는 고립 노드를 해소하여 BFS 탐색이 시작될 수 있도록 최소한의
연결을 확보한다.

→ 스크립트: .manual/scripts/enhance\_wikilinks.py

**7.1 클러스터 링크 주입**

-   같은 tag를 가진 파일들 사이에 상호 링크를 자동 추가

-   각 파일 하단에 \#\# 관련 문서 섹션 생성 후 동일 태그 파일들을
    \[\[wikilink\]\] 형식으로 나열

**7.2 제목 매칭 링크 (Placeholder 방식)**

-   본문 내에 다른 파일의 제목이 텍스트로 등장하면 자동으로
    \[\[wikilink\]\] 변환

-   기존 \[\[\...\]\] 구간은 반드시 보호해야 함 --- placeholder 방식
    필수

> 기존 \[\[\...\]\] 블록 → \\x00WLINK{n}\\x00 placeholder로 치환
>
> → 정규식으로 키워드 교체 → \\x00WLINK{n}\\x00 원복
>
> *Python re 모듈은 가변 길이 lookbehind를 지원하지 않는다. 반드시
> placeholder 방식을 사용할 것. 미적용 시 기존 링크 stem이 오염된다
> (inject\_keywords v1 버그 참조).*
>
> **8. Wiki 링크 2차 강화**

→ 스크립트: .manual/scripts/strengthen\_links.py

**8.1 깨진 브래킷 수정**

> re.sub(r\'\\\[{3,}(\[\^\\\[\\\]\]+?)\\\]{2}\', r\'\[\[\\1\]\]\', text)

**8.2 도메인 허브 링크 주입**

-   특정 엔티티명이 본문에 등장하면 해당 허브 문서 링크 자동 추가

> ENTITY\_HUB = {
>
> \'엔티티명A\': \'허브\_파일\_stem\_A\',
>
> }

**8.3 Ghost → Real 허브 교체**

  -------------------------- ------------------------------------------------
  **교체 전 (ghost)**        **교체 후 (real 허브)**
  \[\[임시\_링크\_A\]\]      \[\[실제\_허브\_문서\|임시\_링크\_A\]\]
  \[\[미정의\_카테고리\]\]   \[\[가장\_관련있는\_파일\|미정의\_카테고리\]\]
  -------------------------- ------------------------------------------------

**8.4 태그 기반 Fallback 링크**

-   위 모든 방법 적용 후에도 링크가 없는 파일 → 같은 태그의 대표 허브
    문서 링크 강제 주입

> TAG\_HUB = {
>
> \'art\': \'아트\_허브\_stem\',
>
> \'tech\': \'기술\_허브\_stem\',
>
> }

> **9. 키워드 링크 주입**

핵심 키워드(캐릭터명, 시스템명 등)가 본문 텍스트로 등장하는 첫 번째
위치를 \[\[허브\_stem\|키워드\]\] 형식으로 자동 교체한다.

→ 스크립트: .manual/scripts/inject\_keywords.py

**9.0 v3 자동 모드 (기본 동작)**

v3부터 KEYWORD\_MAP 수동 등록 없이 완전 자동으로 동작한다.
실행 시 \_index.md를 분석하여 키워드 맵을 동적으로 구성한다.

> python inject\_keywords.py \<vault\_dir\>

자동 키워드 선정 기준 (3단계 필터):

  ---------------------------------- --------------------------------------------------------
  **필터**                           **기준**
  ① stem 등장 수 (MIN\_STEMS)        \_index.md에서 동일 키워드를 포함한 stem ≥ 3개
  ② 평문 등장 파일 수 (MIN\_FREQ)   볼트 전체에서 평문으로 등장하는 파일 ≥ 3개
  ③ 등장 비율 상한 (MAX\_RATE)      볼트 대비 등장 비율 ≤ 15% (범용어 자동 제외)
  ---------------------------------- --------------------------------------------------------

대표 허브 선택 알고리즘: 키워드가 stem의 마지막 의미 토큰인 문서를 허브로
선택한다. 동점이면 토큰 수가 적은 것(더 일반적인 제목)을 우선한다.

> *gen\_index.py 이후 실행이 필수다. inject\_keywords.py를 gen\_index.py보다
> 먼저 실행하면 새로 추가된 문서가 \_index.md에 반영되지 않아 해당
> 문서 관련 키워드가 누락된다.*

**9.1 KEYWORD\_MAP\_MANUAL — 수동 오버라이드**

자동 선정에서 누락되는 항목(약어·별칭·존칭 등)만 수동 등록한다.
수동 항목은 자동 항목보다 우선 적용된다.

> KEYWORD\_MAP\_MANUAL: dict\[str, tuple\[str, str\]\] = {
>
> \# \'키워드\': (\'허브\_파일\_stem\', \'표시\_텍스트\'),
>
> \# 자동 분석에서 누락되는 별칭·약어만 등록
>
> \# \'이사장님\': (\'chief persona(0.1.0)\', \'이사장님\'),
>
> \# \'TLS\': (\'TLS(TimeLineSkill)시스템\_588781620\', \'TLS\'),
>
> }

등록 기준:

  ------------------------------------------ -------------------------------------------------------
  **등록 대상**                              **기준**
  자동 미탐지 별칭·존칭                      vault 내 3개 파일 이상에서 등장
  약어·코드명                                정식 명칭 stem과 함께 등록
  자동 선정 허브가 틀린 경우                 올바른 hub stem으로 오버라이드
  ------------------------------------------ -------------------------------------------------------

  ------------------------------------------ -------------------------------------------------------
  **등록 제외**                              **이유**
  2자 이하 단어                              false positive 과다
  일반 동사·형용사                           허브 연결 의미 없음
  자동 선정이 이미 커버하는 항목             중복 불필요
  ------------------------------------------ -------------------------------------------------------

**9.2 Placeholder 방식 (필수 — v2~v3 공통)**

기존 \[\[\...\]\] 링크 안의 stem이나 display 텍스트에 키워드가 포함되면
오염된다. v2에서 placeholder 방식으로 수정하였다.

> *v1 버그: \[\[07. 캐릭터 \_ 캐릭터A\_432514018\|캐릭터A\]\] 에서
> \'캐릭터A\' 키워드를 교체하면 → \[\[07. 캐릭터 \_ \[\[07. 캐릭터 \_
> 캐릭터A\_432514018\|캐릭터A\]\]\_432514018\|캐릭터A\]\] 처럼 중첩 링크가
> 생성되어 파일이 손상된다.*

수정 내용 (v2·v3 공통):

1.  기존 \[\[\...\]\] 블록 전체를 \\x00WLINK{n}\\x00 placeholder로 치환

2.  placeholder가 적용된 텍스트에만 키워드 정규식 적용

3.  교체 완료 후 placeholder를 원래 링크로 복원

**9.3 주의사항**

-   파일 1개당 키워드 1회만 교체 (최초 등장 위치)

-   코드 블록(\`\`\`) 내 키워드는 건너뜀

-   Frontmatter 영역은 교체 대상에서 제외

-   inject\_keywords.py는 배치 추가 완료 후 1회만 실행 (반복 실행 시 중복 링크 발생)

**9.4 gen\_keyword\_map.py — 진단 도구**

inject\_keywords.py가 내부적으로 사용하는 자동 분석 결과를 미리
확인하거나 KEYWORD\_MAP\_MANUAL 관리에 활용하는 진단 스크립트.
일반적인 파이프라인 실행에서는 별도 호출 불필요.

> python gen\_keyword\_map.py \<vault\_dir\>             \# 미리보기
>
> python gen\_keyword\_map.py \<vault\_dir\> \--apply    \# inject\_keywords.py에 직접 반영

옵션:

  -------------------------------- -------------------------------------------------------
  **옵션**                         **설명**
  \--min-stems N                   최소 stem 수 (기본 3)
  \--min-freq N                    최소 평문 등장 파일 수 (기본 3)
  \--max-rate F                    최대 등장 비율 (기본 0.15)
  \--apply                         inject\_keywords.py KEYWORD\_MAP 자동 갱신
  -------------------------------- -------------------------------------------------------

**9.5 유사어 / 별칭 처리**

같은 허브를 가리키는 표기가 여러 개일 때, 각 표기를 모두
KEYWORD\_MAP\_MANUAL에 등록하고 동일한 허브 stem으로 매핑한다.

> KEYWORD\_MAP\_MANUAL = {
>
> \# 같은 캐릭터의 다양한 표기를 동일 허브로 연결
>
> \'CharacterA\': (\'07. 캐릭터\_캐릭터A\_432514018\', \'캐릭터A\'),
>
> \'붉은 악마\': (\'07. 캐릭터\_캐릭터A\_432514018\', \'붉은 악마\'),
>
> \# 존칭 별칭 — BFS 탐색 엣지 확보를 위해 body 링크 유지 (related: 만으로는 탐색 불가)
>
> \'이사장님\': (\'chief persona(0.1.0)\', \'이사장님\'),  \# 16.4% 등장률이지만 삭제 금지
>
> }

**9.6 범용어 자동 제외 (STOPWORDS)**

inject\_keywords.py 자동 모드는 STOPWORDS를 통해 범용어를 선정 단계에서
제외한다. 새로운 범용어가 자동 선정될 경우 STOPWORDS 집합에 추가한다.

> *판단 기준: **허브에 고유 의미가 있는 고유명사라면 등장률과 무관하게 유지한다.**
> "퀘스트" (41.1%) — 범용어, 허브 연결 의미 없음 → STOPWORDS 추가.
> "이사장님" (16.4%) — chief persona 캐릭터급 허브 → KEYWORD\_MAP\_MANUAL 유지.*
