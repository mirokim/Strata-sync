> **17. 운영 가이드**

**17.1 신규 문서 추가 시**

| # | 작업 | 자동화 가능 여부 |
|---|------|-----------------|
| 1 | Frontmatter 필수 필드 작성 (date·type·status·tags·source·origin) | ✅ `md_normalize.py` 자동 생성 |
| 2 | `## 개요` 섹션에 핵심 내용 요약 (BFS Hop-1 최적화) | ❌ 수동 (내용 판단 필요) |
| 3 | 관련 문서에 `[[wikilink]]` 삽입 + 상위 허브 역방향 링크 추가 | 🔶 `enhance_wikilinks.py` 부분 자동 (클러스터 기반) |
| 4 | 특수 태그(chief 등) 해당 여부 확인 (§11 참조) | ✅ `inject_speaker.py` + chief 태그 자동화 (§11.3.1) |
| 5 | 키워드 역전파 수행 → §17.1.1 참조 | ✅ `inject_keywords.py` |
| 6 | `_index.md` 갱신 → §17.1.2 참조 | ✅ `gen_index.py` 자동 |

> *3번(관련 문서 링크)은 문맥 판단이 필요하여 완전 자동화가 어렵다.
> 나머지 단계는 스크립트 실행으로 대체 가능 — 신규 문서 10개 이상 배치 추가 시 §17.1.4 절차 사용.*
>
> *v3.25: currentSituation.md는 강제 주입 방식이 폐지되어 특별 관리 불필요.*
> *일반 허브 문서로 취급되며 내용이 최신이면 PPR 점수로 자연히 검색 결과에 포함된다.*

**17.1.1 키워드 역전파 절차**

새 문서에 등장하는 핵심 키워드를 기존 문서들에 wikilink로 역전파한다.
새 허브가 생겼을 때 기존 문서들이 그 허브를 가리키도록 연결해야
Graph RAG가 새 문서에 접근하는 경로가 생긴다.

  ------ -----------------------------------------------------------------------
  **순서** **작업**
  ①      gen\_index.py 실행 → \_index.md에 새 문서 반영 (inject\_keywords 전 필수)
  ②      inject\_keywords.py 실행 → \_index.md 자동 분석 후 키워드 맵 구성 및 링크 주입
  ③      변경된 파일 수 및 교체된 키워드 수 확인 (스크립트 출력 참조)
  ④      check\_quality.py 재실행 → 고립 노드 0 확인
  ------ -----------------------------------------------------------------------

> *inject\_keywords.py v3부터 KEYWORD\_MAP 수동 등록 불필요.
> \_index.md 기반으로 자동 분석하여 키워드 맵을 구성한다.
> 자동 선정에서 누락된 키워드(약어·별칭 등)는 KEYWORD\_MAP\_MANUAL에 수동 등록.*
>
> *역전파 후 새 문서에 대한 inbound link가 0이면 상위 허브(연도별 허브 또는
> chief persona)에 수동으로 역링크를 추가한다.*

**17.1.2 \_index.md 갱신 기준**

  ---------------------------------------- -------------------------------------------------------
  **시점**                                 **방법**
  단일 문서 추가 시                        gen\_index.py 즉시 실행
  대량 추가 시 (10개 이상)                 모든 문서 추가 완료 후 1회 실행
  매주 정기 실행                           §17.2 정기 정제 주기 참조
  ---------------------------------------- -------------------------------------------------------

> *\_index.md는 active/ 폴더의 모든 파일 목록을 날짜 역순으로 반영해야 한다.
> 신규 문서가 목록에 없으면 Graph RAG BFS 초기 진입점에서 누락된다.*

**17.1.3 currentSituation.md 관리 (v3.25 변경)**

> ⚠️ **v3.25 변경**: currentSituation.md는 더 이상 BFS 강제 진입점이 아니다.
> PPR 점수 기반으로 일반 허브 문서와 동일하게 검색된다.

내용을 최신으로 유지하면 PPR 점수가 높게 유지되어 관련 쿼리에서 자연히 검색 결과에
포함된다. 내용이 낡아도 검색 품질이 오염되지 않는다 (이전 버전의 강제 주입 버그 해소).

  ---------------------------------------- -------------------------------------------------------
  **반영 권장 대상**                       **기준**
  의사결정 변경이 담긴 문서                기존 방향과 다른 결론이 포함된 경우
  신규 허브 파일                           새로운 주제 클러스터의 진입점
  ---------------------------------------- -------------------------------------------------------

갱신 방법:

- "## 최근 변경사항" 섹션에 `[[새 문서명|한줄 요약]]` 추가
- date 필드를 오늘 날짜로 갱신

> *내용이 2주 이상 미갱신이면 check\_outdated.py가 경고를 출력한다.
> 강제 주입이 없어졌으므로 낡은 내용이 검색 결과에 섞이는 문제는 발생하지 않는다.*

**17.1.4 대량 문서 배치 추가 시**

10개 이상의 문서를 한 번에 추가할 때는 단건 추가와 처리 순서가 다르다.

  ------ -----------------------------------------------------------------------
  **순서** **작업**
  ①      모든 문서 MD 변환 및 frontmatter 작성 완료
  ②      check\_quality.py 실행 → 기준 미달 파일 일괄 수정
  ③      gen\_index.py 실행 → \_index.md 1회 갱신 (inject\_keywords 전 필수)
  ④      inject\_keywords.py 1회 실행 → \_index.md 자동 분석·키워드 주입 (개별 실행 금지)
  ⑤      고립 노드가 많으면 enhance\_wikilinks.py 실행 (§7 참조)
  ⑥      (선택) currentSituation.md에 주요 의사결정 문서 반영
  ⑦      check\_quality.py 재실행 → 전체 품질 지표 최종 확인 (헤딩 비율 99% 이상 확인)
  ------ -----------------------------------------------------------------------

> *inject\_keywords.py v3는 실행 시 \_index.md에서 허브 stem을 자동 추출하여
> 키워드 맵을 동적으로 구성한다. gen\_index.py 이전에 실행하면 새 문서가
> \_index.md에 반영되지 않아 키워드 누락이 발생하므로 순서 준수가 필수다.*
>
> *inject\_keywords.py를 문서 추가마다 반복 실행하면 이미 링크가 있는
> 위치에 중복 링크가 생길 수 있다. 배치 추가 시 반드시 모든 문서 추가 후
> 1회만 실행한다.*

**17.1.5 TF-IDF / BM25 인덱스 자동 재인덱싱**

inject\_keywords.py 실행이나 파일 추가·수정 후 Strata Sync에서 별도로
인덱스를 갱신할 필요가 없다. 볼트를 재오픈하면 파일 mtime 기반
fingerprint를 자동 비교하여 변경이 있을 때만 재인덱싱한다.

  -------------------------------------------- ---------------------------------------------------------------
  **상황**                                     **동작**
  파일 추가 / 수정 / inject 후                 볼트 재오픈 시 fingerprint 불일치 → 자동 재빌드 (수초 소요)
  Edit Agent가 파일 편집 완료 (v3.25)          즉시 IndexedDB 캐시 무효화 → 다음 검색 시 자동 재빌드
  변경 없음                                    IndexedDB 캐시에서 ms 단위 복원
  -------------------------------------------- ---------------------------------------------------------------

→ v3.25부터 Edit Agent 사이클에서 파일이 편집되면 그 즉시 BM25 캐시가 무효화된다.
볼트를 재오픈하지 않아도 다음 검색 요청 시 최신 인덱스가 반영된다.

**17.2 정기 정제 주기**

  ---------- ------------------------------------------------------------------------------------
  **주기**   **작업**
  매주       \_index.md 재생성 (gen\_index.py), (선택) currentSituation.md 의사결정 반영
  매월       주간 보고 병합, 링크 없는 파일 점검, check\_quality.py 실행 (헤딩 99% 확인)
  분기       전체 PageRank 측정, ghost 노드 점검, 빈 문서 정리, 태그 정의서 검토
  ---------- ------------------------------------------------------------------------------------

> 검색을 오염시키지 않는다. 중요한 의사결정 변경 발생 시에만 갱신 권장.*
