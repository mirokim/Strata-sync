> **10. 섹션 헤딩 및 BFS 최적화**

**10.1 섹션 헤딩(\#\#) 추가**

-   Passage-level retrieval: 각 \#\# 섹션이 독립 검색 단위가 됨

-   헤딩 보유 비율 99% 이상 권장 (헤딩 없는 파일은 Passage 검색 대상에서 사실상 제외됨)

-   헤딩 없는 문서: 첫 문단을 \#\# 개요로 감싸는 것만으로도 효과적

**10.2 BFS Hop 최적화 --- 문서 앞부분 배치**

Graph RAG는 BFS로 Hop-1 → Hop-4를 탐색한다. 토큰 한계로 인해 앞
내용일수록 더 많이 읽힌다.

  ------------------- ---------------------------------------
  **Hop 위치**        **배치 권장 내용**
  Hop-1 (가장 중요)   핵심 결론, 현황 요약 (\#\# 개요 섹션)
  Hop-2               관련 허브 문서 링크 모음
  Hop-3               세부 내용, 히스토리
  Hop-4 (참고용)      부록, 참고 자료
  ------------------- ---------------------------------------

**10.3 PPR 컨텍스트 예산 (문서 순위별 읽기 한도)**

Rembrandt Map의 PPR 엔진은 순위별로 아래 글자 수만큼 문서를 읽는다.
이 예산 안에 핵심 정보가 들어 있어야 AI에게 전달된다.

  -------------------------------- ------- -----------------------------------------
  **순위**                         **예산** **비고**
  1--3위 (핵심 문서)               1,500자  directVaultSearch 직접 지목 문서도 동일
  4--8위 (연관 문서)               900자
  9위 이상 (주변 문서)             500자
  -------------------------------- ------- -----------------------------------------

→ 허브 문서의 핵심 내용은 반드시 **상위 1,500자 이내**에 배치. `## 개요` 섹션을
파일 앞부분에 두는 것(§10.2)이 이 예산을 최대한 활용하는 방법이다.

> **11. PageRank 최적화 --- 연도별 허브 구조**

Graph RAG는 높은 PageRank(인바운드 링크 수)를 가진 노드를 우선 탐색한다.
Ghost 노드(파일 없는 링크)가 상위 PageRank를 점유하면 탐색이 의미 없는
노드에서 종료된다.

**11.1 연도별 허브 구조 구축**

대량의 회의록·피드백 문서가 있는 경우 연도별 허브 파일을 생성하여 모든
하위 문서를 연결한다.

> chief persona (최상위 허브)
>
> └── 회의록\_2025 (연도 허브)
>
> ├── \[2025.03.01\] 이사장님 피드백\_XXXXXX
>
> ├── \[2025.05.12\] 이사장님 피드백\_XXXXXX
>
> └── \... (전체 목록)

**11.2 BackLink 확인 방법**

Obsidian Graph View에서 노드 크기로 인바운드 링크 수를 확인한다. 크기가
작은 중요 문서가 있다면 상위 허브에 링크를 추가한다.

**11.3 이사장 피드백 아카이브 패턴**

특정 인물의 피드백 문서가 다수인 경우 아래 패턴으로 계층화한다.

**11.3.1 chief 태그 자동화 로직**

파일명에 아래 키워드 중 하나라도 포함되면 frontmatter tags에 `chief`를
자동 추가한다.

> CHIEF\_KEYWORDS = \[\'이사장\', \'피드백\', \'정례보고\', \'정례 보고\', \'회장님\'\]

자동화 순서:

> 1\. 파일명 기준으로 해당 키워드 포함 파일 필터
>
> 2\. frontmatter의 tags: \[\...\] 줄에 \', chief\' 추가
>
> 3\. 이미 chief가 있는 파일은 스킵

**11.3.2 연도별 허브 파일 자동 생성**

이사장 피드백 파일에서 날짜(\[YYYY.MM.DD\] 패턴)를 추출하여 연도별로
그룹핑한 허브 파일을 자동 생성한다.

> 파일명: `회의록_YYYY.md`
>
> 역방향 링크 형식: 각 피드백 파일 하단에 `> 상위 허브: [[회의록_YYYY]] · [[chief persona]]` 추가

chief persona.md는 최상위 허브로, 연도 허브 전체를 목록으로 연결한다.

  --------------------------- --------------------------------------------------------
  **파일**                    **역할**
  chief persona(0.1.0)        최상위 페르소나 허브. 연도별 허브 링크 목록 포함
  회의록\_20XX.md             연도 허브. 해당 연도 전체 피드백 파일 목록 링크
  \[YYYY.MM.DD\] 피드백\_ID   개별 피드백 문서. type: meeting, tags: \[chief, \...\]
  --------------------------- --------------------------------------------------------

> *개별 피드백 문서에 chief 태그가 누락되면 Graph RAG가 해당 문서를
> 피드백 클러스터에서 찾지 못한다. 파일명에 \'이사장\' \'피드백\'
> \'정례보고\'가 포함된 모든 파일에 chief 태그를 반드시 지정한다.*
>
