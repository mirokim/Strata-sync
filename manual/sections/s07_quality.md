> **12. 이미지 링크 관리**

Obsidian의 !\[\[이미지.png\]\] 형식은 attachmentFolderPath 설정에 따라
실제 파일 경로를 결정한다. 이 경로가 맞지 않으면 이미지가 표시되지 않고
Graph RAG에서 깨진 링크(Broken Node)가 생성된다. 이미지가 원본 문서의
실제 콘텐츠를 포함하는 경우 !\[\[\]\] 링크를 반드시 첨부해야 하며,
파일명은 원본 문서명 기반 규칙을 따른다.

**이미지 파일명 규칙: {문서stem}\_{page}\_{index}.확장자**

-   {문서stem}: Markdown 파일의 stem(확장자 제외 파일명)

-   {page}: 페이지 번호 (p1, p2, \...). HTML 변환 시 생략 가능

-   {index}: 동일 페이지 내 이미지 순번 (1, 2, \...)

-   예시: 기능설계서\_p3\_1.png, 회의록\_2025-03\_1.png

**12.1 .obsidian/app.json 설정**

> {
>
> \"attachmentFolderPath\": \"attachments\",
>
> \"newFileLocation\": \"folder\",
>
> \"newFileFolderPath\": \"active\",
>
> \"alwaysUpdateLinks\": true
>
> }

**12.2 깨진 이미지 링크 처리**

다음 두 가지 방법 중 택일한다.

  ------------------ ---------------------------------------------------------------------------------------------------------------------------------------------
  **방법**           **설명**
  경로 수정 (권장)   attachment 폴더 경로와 일치하도록 이미지 파일 이동 또는 링크 수정. 파일명은 원본 문서명 기반({문서stem}\_{page}\_{index}.png)으로 통일한다.
  링크 제거          !\[\[이미지.png\]\] 전체 삭제. 이미지 파일이 없고 원문에서 단순 장식인 경우에 한함
  ------------------ ---------------------------------------------------------------------------------------------------------------------------------------------

> *이미지가 원본 문서의 실제 콘텐츠를 포함하는 경우 반드시 attachments/
> 폴더에 파일을 두고 !\[\[\]\] 링크를 유지한다. attachments/ 폴더에 실제
> 이미지 파일이 없으면서 !\[\[\]\] 링크가 있으면 Graph RAG에서 Phantom
> Node를 생성하여 PageRank를 왜곡한다. 이 경우에만 audit\_and\_fix.py
> \--fix 로 링크를 제거한다.*
>

**13.1.1 graph\_weight 분포 확인**

**13.1.2 고아 첨부 파일 점검**

생성된 임시 테스트 파일(`test_*.png` 등)이나 archive 이동 후 남겨진 파일이
주요 대상이다.

> 처리 기준:
>
> \- 즉시 삭제가 확실한 경우 → 삭제
>
> \- 확인이 필요한 경우 → `attachments/_orphan/` 폴더로 격리 보관

> *`attachments/_orphan/` 폴더는 Graph RAG 탐색에 영향을 주지 않는다.
> Obsidian의 attachmentFolderPath 설정은 루트 attachments/ 만 인식하기 때문이다.
> 일정 기간 후 내용을 확인하고 최종 삭제 여부를 결정한다.*

**13.2.1 괄호 포함 파일명 링크 절단 버그**

파일명에 소괄호 `()`가 포함된 경우 일부 파서에서 `)` 이후 텍스트가
wikilink stem에서 절단되는 버그가 발생한다.

  ----------------------------------------------- -------------------------------------------
  **절단된 링크 (broken)**                         **실제 파일 stem**
  `[[TLS(TimeLineSkill]]`                          `TLS(TimeLineSkill)시스템`
  `[[MTP(maptoolplus]]`                            `MTP(maptoolplus) 가이드`
  ----------------------------------------------- -------------------------------------------

자동 복구 방법: broken link stem이 기존 파일 stem의 prefix(접두사)와
일치하면 자동으로 실제 stem으로 교체한다.

> candidates = \[s for s in existing\_stems if s.startswith(broken\_stem)\]
>
> → 후보가 1개이면 자동 교체, 2개 이상이면 가장 짧은 것 선택

**13.3 MANUAL\_MAP 활용**

자동 해결 불가한 broken link (예: 다른 폴더의 문서, 외부 시스템 링크)는
MANUAL\_MAP에 수동 등록한다.

> MANUAL\_MAP: dict\[str, str\] = {
>
> \# \'broken\_stem\': \'real\_stem\',
>
> \'TLS\': \'TLS(TimeLineSkill)시스템\_588781620\',
>
> \# ── 날짜 대괄호 형식 불일치: \"2026.01.07 \...\" → \"\[2026.01.07\]
> \...\"
>
> \'2026.01.07 피드백\_ID\': \'\[2026.01.07\] 피드백\_ID\',
>
> \# ── 따옴표 포함 파일명: 링크에 따옴표 있지만 파일명은 언더스코어
> 치환
>
> \'\"quoted title\"\': \'\_quoted\_title\_\_ID\',
>
> \# ── 마크다운 테이블 내 백슬래시 이스케이프: \[\[stem\\\_suffix\]\]
> 패턴
>
> \'문서명\\\\\_suffix\': \'문서명\_suffix\_ID\',
>
> }

**13.4 3중 대괄호 false positive 패턴**

Obsidian 파일명에 \[범주\] 접두사 패턴(예: \[기획\] 파일명,
\[2025.01.01\] 날짜)이 사용되면 wikilink가 \[\[\[범주\] stem\]\] 형태로
나타난다. 이는 유효한 Obsidian 링크이므로 수정 대상이 아니다.

  ----------------------------------- -------------------------------------
  **패턴**                            **판단**
  \[\[\[2025.01.01\] 피드백\_ID\]\]   날짜형 접두사. Obsidian이 정상 파싱
  \[\[\[기획\] 기능명\_ID\]\]         범주형 접두사. Obsidian이 정상 파싱
  \[\[\[\[stem\]\]                    4중 이상. 실제 버그 --- 수정 필요
  ----------------------------------- -------------------------------------
