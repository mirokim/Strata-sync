## 수정 사항 (이번 세션)

### 1. directVaultSearch 스코어링 전면 개편 (`graphRAG.ts`)
- **Before**: `filenameWeight(10)` × hitCount + `bodyWeight(1)` × hitCount → `score / (terms.length * filenameWeight)` 정규화
  - 단일 파일명 매칭 → 1.0 점수, 모든 문서가 0.9~1.0 클러스터링
  - `titleHit` 절대 우선 정렬 (파일명 1개 매칭이 본문 전체 매칭보다 우선)
- **After**: 커버리지 비율 기반 — `(filenameHits/n) * 0.6 + (bodyHits/n) * 0.4`
  - 파일명 매칭 부스트: `0.5 + score * 0.5` (0.5~1.0 범위, BM25과 경쟁 가능)
  - `titleHit` 절대 우선 제거 → 순수 점수 기반 정렬

### 2. BM25 query term coverage 보정 (`graphAnalysis.ts`)
- **Before**: raw BM25 score만 사용 → 주간업무보고/정례보고 같은 짧은 잡다 문서가 범용 키워드 1~2개만 매칭해도 고점
- **After**: `rawScore * coverage^0.5` 보정 (coverage = matchedTerms / queryTermCount)
  - 1/3 매칭: ×0.58 감점 / 2/3 매칭: ×0.82 / 3/3: ×1.0
  - 주간업무보고의 범용어 과잉 매칭 억제
- TFIDF_SCHEMA_VERSION: 5 → 6

### 3. 테스트 머지 전략 개선 (`searchScenarios*.test.ts`)
- **Before**: direct hits 먼저 추가 → BM25 추가 (중복 제거) → 점수 정렬
- **After**: 양쪽 파이프라인 점수 맵 구축 → dual-hit 보너스 (`max + min * 0.3`)
  - 두 파이프라인 모두 발견한 문서에 30% 추가 부스트

### 4. Recency boost — 최근 문서 가산 (`graphRAG.ts`, `graphAnalysis.ts`)
- **수식**: `score *= 1 + 0.1 * exp(-daysOld / 180)`
  - 6개월 이내: ~10% 부스트 / 1년 경과: ~3% / 2년+: 거의 0
- **파일명 날짜 파싱** (`parseFilenameDate`): `[YYYY_MM_DD]`, `YYYYMMDD`, `YY_MM_DD`, `YYMMDD` 패턴 지원
  - 파일시스템 mtime은 동기화 시점이라 무의미 → 파일명 > frontmatter date > mtime 우선순위
  - 연도 2000~2039 제한으로 Notion ID 오파싱 방지, 미래 30일 이내만 허용
- **적용 위치**: directVaultSearch + BM25 search 양쪽 동시 적용
- TFIDF_SCHEMA_VERSION: 6 → 7
- **효과**: "이사장님 최근 피드백" 2026-01-07 문서 5위→1위, "2026년 1월 작업" 26-01 문서 상위 진입
