# 검색 알고리즘 설명

> 최종 수정: 2026-03-27

---

## 전체 흐름 요약

```
사용자 쿼리
    │
    ▼
[0] 전체 탐색 인텐트? ──Yes──▶ 허브 노드 BFS (그래프 전체 탐색) ──▶ END
    │ No
    ▼
[1] 파일명 직접 매칭 (filenameWeight=10)
    │
    ├─ score ≥ 0.4 (강한 매칭) ──▶ 전체 본문 직접 주입 + BFS 보완 ──▶ END
    │
    ├─ score ≥ 0.2 (약한 매칭) ──▶ BFS 시드로만 사용 ──▶ [3]
    │
    └─ 매칭 없음 ──▶ [2]
    │
    ▼
[2] BM25 키워드 검색 + 벡터 Reranking (폴백)
    │
    ▼
[3] Rerank → BFS 그래프 탐색 → 컨텍스트 조립 ──▶ END
```

---

## 단계별 상세 설명

### [0] 전체 탐색 인텐트 감지

쿼리에 "전체", "모든", "다 알려줘" 같은 패턴이 포함되면 키워드 검색을 건너뜁니다.
대신 **PageRank 기반 허브 노드** 상위 35개를 중심으로 BFS 깊이 4로 그래프를 탐색해 광범위한 컨텍스트를 수집합니다.

---

### [1] 파일명 직접 매칭

볼트 전체 문서의 **파일명**에 쿼리 단어가 포함되는지 확인합니다.

| 조건 | 점수 계산 | 처리 방식 |
|------|-----------|-----------|
| 쿼리 단어 2개 이상 파일명 매칭 | score ≥ 0.4 | **강한 매칭** — 해당 파일 본문 전체를 즉시 주입 |
| 쿼리 단어 1개 파일명 매칭 | 0.2 ≤ score < 0.4 | **약한 매칭** — BFS 시드로만 사용 |

**점수 계산 공식:**
```
파일명 히트 점수 = (파일명에 매칭된 단어 수) × filenameWeight(10)
본문 히트 점수  = (본문에 매칭된 단어 수)   × bodyWeight(1)
최종 score = (파일명 히트 + 본문 히트) / 정규화 인수
```

강한 매칭이 발견되면 BFS로 wiki-link 연결 문서도 추가로 보완합니다.

---

### [2] BM25 키워드 검색 + 벡터 Reranking (폴백)

직접 매칭이 부족할 때 실행됩니다.

#### 2-1. BM25 후보 추출

백엔드(ChromaDB)가 있으면 사용, 없으면 **프론트엔드 BM25 인덱스**로 폴백합니다.
벡터 임베딩 인덱스가 빌드된 경우 더 많은 후보(`bm25Candidates × 4`)를 뽑아 reranking 헤드룸을 확보합니다.

#### 2-2. 벡터 Hybrid Reranking (Gemini 임베딩)

Gemini `gemini-embedding-001` 모델 (3072차원)을 사용합니다.

```
최종 점수 = BM25 정규화 점수 × 0.4 + 코사인 유사도 × 0.6
```

| 구성 요소 | 역할 |
|-----------|------|
| BM25 (40%) | 키워드 정확도 — "캐릭터C 스킬" 같은 명확한 단어 매칭 |
| 벡터 유사도 (60%) | 의미 유사도 — "멍멍이" → "강아지" 같은 의미 기반 매칭 |

임베딩 인덱스가 없으면 순수 BM25 결과를 사용합니다.

#### 2-3. Recency Boost

최신 인텐트("최신", "최근", "요즘") 감지 여부에 따라 최근 문서에 보너스 점수를 부여합니다.

```
recency 점수 = recencyCoeff × exp(-(현재 - 문서날짜) / halfLife)
```

| 파라미터 | 기본값 | 설명 |
|----------|--------|------|
| `recencyHalfLifeDays` | 180일 | 이 기간 지나면 점수가 절반으로 |
| `recencyCoeffNormal` | 0.4 | 일반 쿼리 |
| `recencyCoeffHot` | 2.0 | "최신/최근" 인텐트 쿼리 |

---

### [3] Rerank → BFS → 컨텍스트 조립

#### Reranking (키워드 + 화자 친화도)

```
seeds = rerankResults(candidates, query, rerankSeeds=5, currentSpeaker)
```

화자(페르소나)가 지정된 경우 해당 화자 문서에 추가 가중치를 줍니다.

#### BFS 그래프 탐색

seeds 문서에서 wiki-link(`[[문서명]]`)를 따라 연결 문서를 탐색합니다.

```
BFS 시드(seeds) → 이웃 문서 탐색 → 관련도 높은 문서 추가
```

#### 컨텍스트 조립 우선순위

1. `_index.md` — 항상 포함 (볼트 전체 구조 파악용)
2. 강한 파일명 매칭 문서 (전체 본문)
3. BFS/Hybrid 검색 결과
4. 그래프 이웃 문서 (wiki-link 연결)

## 파라미터 요약표

| 파라미터 | 기본값 | 설명 |
|----------|--------|------|
| `filenameWeight` | 10 | 파일명 매칭 점수 배율 |
| `bodyWeight` | 1 | 본문 매칭 점수 배율 |
| `directCandidatesNormal` | 20 | 일반 쿼리 직접 검색 후보 수 |
| `directCandidatesRecency` | 50 | 최신 인텐트 직접 검색 후보 수 |
| `directHitSeeds` | 8 | BFS 시드로 사용할 직접 검색 상위 N개 |
| `bm25Candidates` | 8 | BM25 폴백 후보 수 |
| `rerankSeeds` | 5 | Rerank 후 최종 시드 수 |
| `minDirectHitScore` | 0.2 | 직접 매칭 유효 임계값 |
| `minPinnedScore` | 0.4 | 전체 본문 직접 주입 임계값 |
| `minBm25Score` | 0.05 | BM25 최소 유효 점수 |
| `rerankVectorWeight` | 0.6 | 벡터 유사도 가중치 (BM25는 1-0.6=0.4) |
| `recencyHalfLifeDays` | 180 | Recency 반감기 |
| `recencyCoeffNormal` | 0.4 | 일반 쿼리 recency 계수 |
| `recencyCoeffHot` | 2.0 | 최신 인텐트 쿼리 recency 계수 |

> 설정 › 검색 튜닝 탭에서 런타임 조정 가능합니다.

## 문서 임베딩 캐시

벡터 임베딩은 볼트 로드 시 백그라운드에서 자동 빌드되며 **IndexedDB**에 캐시됩니다.

```
볼트 로드
    │
    ▼
IndexedDB 캐시 존재? ──Yes──▶ 즉시 복원 (API 호출 없음)
    │ No
    ▼
Gemini API 배치 호출 (20개/배치, 5개 병렬)
    │
    ▼
IndexedDB 저장 (다음 로드부터 캐시 사용)
```

캐시 키: `vaultPath + 볼트 지문(fingerprint)` — 볼트 파일이 변경되면 자동 재빌드.

문서당 텍스트 한도: **3000자** (파일명 + 태그 + 화자 + 섹션 내용 순서로 조합)
