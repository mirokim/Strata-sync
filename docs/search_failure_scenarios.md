## 1. 한국어 토크나이저 (`tokenize`, `stemKorean`)

### 1.1 [실패] 단일 문자 토큰 필터링 — "AI", "3월" 검색 불가

- **현상**: 쿼리 "AI 활용" → 토큰 ["활용"]. "AI"는 2글자지만 영문 tokenize 후 소문자화 "ai"가 filter에서 통과되는 경우와 안 되는 경우가 불일치
- **원인**: `filter(t => t.length > 1)`에서 1자 토큰 제거. CJK 1자("봇", "앱")도 소실
- **영향**: 짧은 약어/고유명사 검색 실패

### 1.2 [부정확] 조사 제거 1회 한정 — 복합 조사 미처리

- **현상**: "이사장님의" → "이사장님" (OK), "받고있는" → "받고" (1회만 제거, "있는" 잔존 안 됨)
- **원인**: `stemKorean()`이 접미사 리스트 순회 후 첫 매칭에서 `break`
- **영향**: 복합 조사가 붙은 쿼리의 검색 정확도 저하

### 1.3 [부정확] 접미사 리스트 정적 — 새 조사 패턴 미반영

- **현상**: "~스럽게", "~답게" 등 비표준 접미사가 제거되지 않음
- **원인**: 하드코딩된 접미사 배열
- **영향**: 구어체 쿼리에서 매칭 실패

### 1.4 [실패] CJK 띄어쓰기 없는 일본어/중국어 토큰화

- **현상**: "機動戦士" → ["機", "動", "戦", "士"] → 모두 1자 → 전부 필터됨 → 결과 0건
- **원인**: 공백 기반 분할만 사용, 형태소 분석 없음
- **영향**: 일본어/중국어 혼합 문서 검색 불가

### 1.5 [부정확] URL/경로 파편화

- **현상**: 문서에 `https://example.com/path`가 있을 때 → ["https", "example", "com", "path"]. "example.com" 검색 시 개별 토큰 매칭은 되지만 연속 매칭 불가
- **원인**: 구두점/슬래시로 분할
- **영향**: URL 검색 시 관련 없는 문서가 높은 점수

### 1.6 [부정확] 숫자+단위 파싱 불완전

- **현상**: "28일" → ["28", "일"] OK, "28개" → 단위 리스트에 "개" 없으면 "28개" 통째로 → 매칭 실패
- **원인**: 숫자+단위 정규식의 단위 리스트 불완전
- **영향**: 수량 관련 쿼리 부정확

### 1.7 [부정확] 영문 대소문자 정규화 후 원본 매칭 불일치

- **현상**: 문서에 "OAuth 2.0 PKCE" → 토큰 ["oauth", "2", "0", "pkce"]. 쿼리 "PKCE" → "pkce". 매칭은 되지만 하이라이트/스니펫에서 원본 위치 추적 불가
- **영향**: 검색은 되지만 결과 미리보기가 정확한 위치를 못 찾음

### 1.8 [실패] 특수문자 포함 키워드 — 괄호, 하이픈

- **현상**: "TLS(TimeLineSkill)" → 괄호에서 분할 → ["tls", "timelineskill"]. 정확한 함수명 검색 불가
- **원인**: 괄호가 분할 구분자
- **영향**: 코드명/시스템명 정확 검색 실패

## 2. BM25/TF-IDF 인덱스 (`graphAnalysis.ts`)

### 2.1 [부정확] IDF 값 증분 업데이트 시 재계산 안 됨

- **현상**: 새 문서 추가 후 `updateDoc()` → 기존 단어의 IDF가 변하지 않음 (코퍼스 크기 변경 미반영)
- **원인**: `idfVal = this.idf.get(term) ?? 1`에서 기존 값 재사용
- **영향**: 새 문서에만 있는 희귀 단어가 과소평가, 흔한 단어가 과대평가

### 2.2 [실패] 빈 벡터 문서 — 암묵적 링크 검출 누락

- **현상**: BM25 벡터 norm이 0인 문서 → `findImplicitLinks()`에서 무조건 건너뜀
- **원인**: `if (a.bm25Norm === 0 || b.bm25Norm === 0) continue`
- **영향**: 이미지만 있는 문서, frontmatter만 있는 문서가 그래프에서 완전 고립

### 2.3 [부정확] 코사인 유사도 임계값 고정 (0.25)

- **현상**: 좁은 도메인(예: 모두 "게임 디자인" 관련)에서 대부분 문서가 0.2~0.3 유사도 → 임계값 0.25 미만으로 링크 누락
- **원인**: `findImplicitLinks(threshold = 0.25)` 하드코딩
- **영향**: 도메인 특화 볼트에서 암묵적 링크가 과소 생성

### 2.4 [오류] 인덱스 빌드 전 검색 호출

- **현상**: 볼트 로드 직후 검색 → `tfidfIndex.isBuilt === false` → TF-IDF 스킵, 키워드 폴백만 작동
- **원인**: 인덱스 빌드가 비동기, 완료 전 검색 가능
- **영향**: 첫 검색이 저품질 키워드 매칭 결과

### 2.5 [부정확] `updateDoc()` 후 캐시 잔존

- **현상**: 문서 수정 후 `updateDoc()` → `_implicitLinks` 캐시가 갱신 안 됨 → 이전 유사도 기반 링크 잔존
- **원인**: 캐시 무효화가 reference equality 의존
- **영향**: 수정된 문서의 그래프 관계가 이전 상태 유지

### 2.6 [부정확] graph_weight: skip 문서가 검색에 포함

- **현상**: raw Jira 이슈(graph_weight: skip)가 BM25 인덱스에 포함되어 검색 상위 점령
- **원인**: 인덱스 빌드 시 graph_weight 필터링 미적용 (그래프에서만 제외)
- **영향**: 500개 얇은 Jira raw 이슈가 BM25 상위 점령 → 노이즈 폭발

### 2.7 [실패] 코퍼스 0건에서 검색

- **현상**: 빈 볼트 또는 로드 실패 후 검색 → 빈 배열 반환, 사용자에게 "결과 없음"
- **원인**: `loadedDocuments.length === 0` 조기 반환
- **영향**: 볼트 로드 실패를 사용자가 "검색 결과 없음"으로 오해

### 2.8 [부정확] 문서 삭제 후 인덱스에 잔류

- **현상**: 파일 삭제 → `updateDoc()` 호출 안 됨 → 삭제된 문서가 검색 결과에 나타남
- **원인**: 증분 업데이트는 수정만 처리, 삭제는 전체 리빌드 필요
- **영향**: 존재하지 않는 문서가 검색 결과에 나옴

### 2.9 [부정확] 섹션 단위 검색에서 빈 섹션 포함

- **현상**: 문서의 빈 섹션(heading만 있고 body 없음)이 best section으로 선택
- **원인**: `doc.sections.find(s => s.body.trim())`가 첫 비어있지 않은 섹션을 찾지만, `queryStems.length === 0`이면 무조건 첫 섹션 반환
- **영향**: 스니펫이 비어있는 검색 결과 표시

## 3. IndexedDB 캐시 (`tfidfCache.ts`)

### 3.1 [오류] 스키마 버전 변경 시 무조건 캐시 미스

- **현상**: 앱 업데이트 → `TFIDF_SCHEMA_VERSION` 증가 → 모든 사용자의 캐시 무효화 → 첫 검색에서 10초+ 리빌드
- **영향**: 업데이트 직후 사용자가 검색 행(hang) 경험

### 3.2 [오류] IndexedDB 저장 실패 무시

- **현상**: IndexedDB 쓰기 실패(용량 초과, 권한) → 경고 로그만 → 다음 세션에서 다시 전체 리빌드
- **원인**: `saveTfIdfCache()` catch에서 경고만 출력
- **영향**: 매 세션마다 느린 첫 검색 반복

### 3.3 [오류] 핑거프린트 충돌

- **현상**: 파일명에 `|` 포함 시 핑거프린트 파싱 오류 → 캐시 항상 미스
- **원인**: `document.id:mtime` 구분자와 `|` join 구분자 충돌
- **영향**: 특정 파일명 때문에 전체 캐시 무효화

### 3.4 [오류] mtime null → 핑거프린트 0

- **현상**: mtime이 null인 문서 2개 → 핑거프린트 동일 → 하나가 캐시에서 덮어씀
- **영향**: 한 문서의 인덱스 데이터가 다른 문서 것으로 교체

### 3.5 [오류] 멀티 탭 동시 접근

- **현상**: 브라우저 탭 2개에서 같은 볼트 열기 → IndexedDB 동시 읽기/쓰기 → 트랜잭션 미사용으로 데이터 손상 가능
- **영향**: 인덱스 깨짐 → 검색 결과 무작위

### 3.6 [실패] Private/Incognito 모드에서 IndexedDB 차단

- **현상**: 일부 브라우저(Safari, Firefox strict)에서 Private 모드 시 IndexedDB 접근 불가 → 매 세션 전체 리빌드
- **영향**: 매번 느린 첫 검색

## 4. Direct Vault Search (파일명 매칭)

### 4.1 [부정확] 파일명 가중치 10배 — 제목 편향

- **현상**: "회의" 검색 → "[회의] 2026.md" 파일이 본문에 "회의"가 50번 등장하는 실제 관련 문서보다 항상 상위
- **원인**: `filenameWeight=10, bodyWeight=1`
- **영향**: 파일명에 키워드가 있는 문서가 무조건 상위 → 실제 관련 문서 매몰

### 4.2 [실패] 날짜 형식 불일치 — "28일" vs "[2026.01.28]"

- **현상**: "28일" → 토큰 ["28", "일"]. 파일명 "[2026.01.28]" → ["2026", "01", "28"]. "28" 매칭되지만 "일"은 미매칭 → 낮은 점수
- **원인**: 날짜 토큰화 불일치
- **영향**: 날짜 기반 검색 정확도 저하

### 4.3 [부정확] 점수 정규화에서 단일 이상치 지배

- **현상**: 1건의 완벽 매칭(score 1.0) → 나머지 모든 결과의 정규화 점수 압축
- **원인**: `maxScore = max(scores) || 1` 정규화
- **영향**: 상위 1건 외 나머지 결과 간 점수 차이 소실

### 4.4 [실패] 파일명 없는 문서 — id만 있는 경우

- **현상**: 프로그래밍적으로 생성된 문서(id는 있지만 filename 없음) → direct search에서 누락
- **영향**: 자동 생성 문서(집계 등) 검색 불가

### 4.5 [부정확] 조사가 파일명에 포함된 경우

- **현상**: 파일명 "시스템의_설계.md" → 토큰 ["시스템의", "설계"]. 쿼리 "시스템" → "시스템" vs "시스템의" 미매칭
- **원인**: 파일명 토큰에 조사 제거 미적용
- **영향**: 파일명에 조사가 있는 문서 검색 누락

### 4.6 [실패] 대소문자 혼합 파일명

- **현상**: 파일명 "OAuth2_PKCE.md", 쿼리 "oauth" → lowercase 매칭 성공. 하지만 `"PKCE"` 단독 쿼리 → 2자 필터? 아니 4자이니 OK. 다만 원본 케이스 복원 시 하이라이트 위치 불일치
- **영향**: 검색은 되지만 UI 하이라이트 깨짐

### 4.7 [부정확] Frontmatter 내용이 검색 대상에 포함

- **현상**: frontmatter의 `tags: [jira, epic, auth]` → "jira" 검색 시 모든 Jira 문서 매칭 (본문 관련성 무시)
- **영향**: 메타데이터 키워드가 본문 관련성을 왜곡

## 5. Frontend Keyword Search

### 5.1 [실패] TF-IDF 미빌드 시 무경고 폴백

- **현상**: 인덱스 빌드 전 검색 → 키워드 폴백 → 사용자는 "TF-IDF 미적용" 사실을 모름
- **영향**: 사용자가 저품질 결과를 정상으로 오인

### 5.2 [부정확] Speaker Affinity 부스트 누락

- **현상**: 커스텀 페르소나 ID가 `PERSONA_TAG_MAP`에 없음 → 부스트 0
- **원인**: 5개 내장 페르소나만 등록 (chief/art/plan/level/prog)
- **영향**: 커스텀 페르소나 사용 시 speaker 기반 필터링 불가

### 5.3 [부정확] outdated 문서 패널티 부족

- **현상**: `outdatedPenalty = -0.25` → 높은 TF-IDF 점수의 outdated 문서가 여전히 상위
- **원인**: 패널티가 고정값이라 높은 기본 점수를 상쇄 못 함
- **영향**: 구버전 문서가 최신 문서보다 상위 랭크

### 5.4 [부정확] 검색 결과 중복

- **현상**: 같은 문서가 TF-IDF 결과와 Direct Search 결과에 모두 포함 → 중복 표시
- **원인**: `[...tfIdfResults, ...fallbackResults]` 병합 후 dedup 미처리
- **영향**: 같은 문서가 검색 결과에 2번 표시

### 5.5 [부정확] 섹션 선택 시 빈 body 우선

- **현상**: 문서의 첫 섹션(## 제목만, body 없음)이 best section으로 선택
- **원인**: `queryStems.length === 0`이면 `bestSection = doc.sections.find(s => s.body.trim())` → 첫 비빈 섹션이지만, 모든 섹션이 빈 경우 undefined
- **영향**: 스니펫 없는 검색 결과

## 6. Reranking (`rerankResults`)

### 6.1 [부정확] 키워드 오버랩 — 어순 무시

- **현상**: 쿼리 "전투 시스템 설계" vs 결과 "설계 시스템 전투" → 동일 점수
- **원인**: 단순 키워드 카운팅, 어순/인접성 미고려
- **영향**: 의미적으로 다른 문서가 동일 순위

### 6.2 [부정확] Speaker 부스트 미약 (+0.1)

- **현상**: Speaker 매칭 부스트 0.1 → TF-IDF 점수 차이 0.15만 돼도 역전
- **원인**: 가산 방식(additive), 승산 방식(multiplicative) 아님
- **영향**: 페르소나 검색에서 담당자 문서가 상위에 못 옴

### 6.3 [부정확] topN 미적용 엣지 케이스

- **현상**: 결과가 topN 이하면 정렬만 하고 전부 반환 → 저품질 결과 포함
- **원인**: `if (results.length <= topN) return sorted` 조기 반환
- **영향**: 관련 없는 문서가 결과에 포함

### 6.4 [부정확] 재랭킹에서 원본 TF-IDF 점수 무시

- **현상**: 재랭킹 시 키워드 오버랩만으로 순위 재결정 → TF-IDF의 정교한 점수가 무시될 수 있음
- **영향**: TF-IDF가 정확히 찾은 결과가 재랭킹에서 밀림

### 6.5 [부정확] 긴 문서 편향

- **현상**: 본문이 긴 문서일수록 키워드 오버랩 확률 높음 → 짧지만 정확한 문서보다 높은 점수
- **원인**: 정규화 없는 키워드 카운팅
- **영향**: 길고 포괄적인 문서가 짧고 정확한 문서를 밀어냄

## 7. Graph Expansion (BFS)

### 7.1 [부정확] WikiLink 양방향 가정

- **현상**: A→B 단방향 링크를 양방향으로 취급 → B에서 A로 역방향 탐색 가능
- **원인**: `buildAdjacencyMap()`이 `adj.get(target)!.push(source)` 추가
- **영향**: 의도하지 않은 역방향 탐색 → 관련 없는 문서 컨텍스트에 포함

### 7.2 [오류] maxDocs 초과 반환

- **현상**: BFS 레벨 단위 탐색 → 마지막 레벨에서 maxDocs 초과해도 레벨 완료까지 수집
- **원인**: 레벨 완료 후 검사
- **영향**: 예상보다 많은 문서가 컨텍스트에 포함 → 토큰 예산 초과

### 7.3 [실패] 고립 노드에서 BFS 시작

- **현상**: 링크 없는 문서가 시드 → adjacency에 없음 → BFS 확장 0건
- **영향**: 고립 문서 관련 질문에 그래프 컨텍스트 없음

### 7.4 [부정확] 삭제된 문서 ID가 adjacency에 잔존

- **현상**: 문서 삭제 → adjacency 캐시 미갱신 → BFS가 존재하지 않는 노드 방문 → null 룩업
- **원인**: 캐시 무효화 타이밍
- **영향**: 삭제된 문서로의 팬텀 경로 탐색

### 7.5 [부정확] Hub 노드를 통한 과도한 확장

- **현상**: _index.md (인바운드 100+) → BFS Hop-1에서 100개 문서 방문 → 대부분 관련 없음
- **원인**: 허브 노드의 높은 degree를 제한하지 않음
- **영향**: 관련 없는 문서가 컨텍스트 오염

### 7.6 [실패] 섹션 ID → 문서 매핑 실패

- **현상**: `sectionMap.get(section_id)` 반환 undefined → 해당 섹션 컨텍스트 누락
- **원인**: 문서 리로드 후 section ID 재생성 → 이전 ID와 불일치
- **영향**: BFS 결과 중 일부 섹션 내용 소실

## 8. PPR (Personalized PageRank)

### 8.1 [오류] Worker 타임아웃 없음 — 무한 대기

- **현상**: PPR Worker가 행(hang) → `await runPPRInWorker()` 무한 대기
- **원인**: Worker에 타임아웃 미설정
- **영향**: 검색이 영원히 완료되지 않음 → 사용자가 앱 재시작 필요

### 8.2 [오류] Worker 크래시 시 전체 요청 실패

- **현상**: Worker onerror → 대기 중인 **모든** PPR 요청이 동일 에러로 reject
- **원인**: 에러 핸들러가 모든 pending promise를 reject
- **영향**: 동시 검색 중 하나의 오류가 전부 실패시킴

### 8.3 [부정확] Dangling 노드 가중치 처리

- **현상**: outgoing edge 없는 노드 → `totalW = outWeightSum.get(from) ?? 1` → 실제 0인데 1로 처리
- **원인**: Null coalescing 기본값
- **영향**: Dangling 노드의 PPR 점수 왜곡

### 8.4 [부정확] Seed 노드 이중 부스트

- **현상**: Seed 노드가 매 iteration마다 `alpha * seedVal` 부스트 + 다른 노드에서 전파받는 점수
- **원인**: PPR 알고리즘 구현 상의 이중 가산
- **영향**: Seed 노드 과도하게 높은 점수 → 다양성 부족

### 8.5 [실패] 빈 그래프에서 PPR 실행

- **현상**: 링크 0건 → 모든 노드가 dangling → PPR 수렴 안 됨 or 균등 분포
- **영향**: PPR 점수가 문서 관련성을 반영 못 함

### 8.6 [부정확] graph_weight 미반영

- **현상**: graph_weight: low (0.3 감쇄) 또는 skip (제외) 설정이 PPR 계산에 반영 안 될 수 있음
- **원인**: PPR Worker가 link weight를 수신하지만 적용 로직 검증 필요
- **영향**: skip 문서가 PPR에서 높은 점수 획득

## 9. RAG Context Building (`fetchRAGContext`)

### 9.1 [오류] 예외 시 빈 컨텍스트 무경고 반환

- **현상**: Stage 1/2에서 예외 → `catch { return '' }` → AI가 RAG 없이 응답
- **원인**: 에러를 로그만 하고 빈 문자열 반환
- **영향**: 사용자는 AI가 볼트를 참조하지 않았다는 사실을 모름

### 9.2 [부정확] 컨텍스트 예산 초과 시 잘림

- **현상**: PPR 상위 문서의 합산 길이가 토큰 예산 초과 → 뒤쪽 문서 잘림
- **원인**: 순위별 글자 수 예산 (1,500 / 900 / 500)
- **영향**: 중요하지만 순위 낮은 문서의 내용 소실

### 9.3 [부정확] 멀티 볼트 검색 시 볼트 간 점수 비교 불가

- **현상**: 볼트 A의 점수 0.8과 볼트 B의 점수 0.6 → A가 우선이지만, 코퍼스 크기가 다르므로 직접 비교 무의미
- **원인**: 볼트별 독립 인덱스, 점수 정규화 미통일
- **영향**: 작은 볼트의 문서가 과대평가

### 9.4 [오류] Worker Summary 실패 시 인덱스 불일치

- **현상**: 3개 문서 요약 중 2번째 실패 → `[summary1, null, summary3]` → 필터 후 [summary1, summary3] → 원본 hit 인덱스와 불일치
- **영향**: 잘못된 문서의 요약이 다른 문서에 매칭

### 9.5 [부정확] 이전 검색 결과와 현재 결과 혼합

- **현상**: 빠른 연속 검색 시 이전 `fetchRAGContext` 비동기 결과와 현재 결과가 경쟁
- **원인**: 동시성 제어 부족
- **영향**: 이전 질문의 컨텍스트가 현재 답변에 사용

### 9.6 [실패] overrideRagContext 사용 시 검증 없음

- **현상**: 외부에서 전달된 overrideRagContext가 오래된 정보 → 그대로 AI에 전달
- **원인**: override 시 검증/갱신 로직 없음
- **영향**: 오래된 컨텍스트 기반 AI 응답

### 9.7 [부정확] 그래프 하이라이트와 컨텍스트 비동기

- **현상**: 그래프 노드 하이라이트가 컨텍스트 빌드 완료 전에 표시 → 실제 사용된 문서와 하이라이트 불일치
- **원인**: `setTimeout(r, 0)` yield가 UI 렌더링 보장 안 함
- **영향**: 사용자가 참조된 문서를 시각적으로 잘못 판단

### 9.8 [실패] 볼트 로드 실패 후 검색 시도

- **현상**: 볼트 파일 읽기 실패 → `loadedDocuments = []` → 검색 결과 0건
- **원인**: 볼트 로드 에러가 검색 레이어까지 전파 안 됨
- **영향**: "검색 결과 없음" vs "볼트 로드 실패" 구분 불가

## 10. 날짜 · 최신성 (Recency)

### 10.1 [부정확] 최신성 쿼리 감지 정규식 편향

- **현상**: "최신 상태" → 감지 OK. "상태가 최신인" → 미감지 (어순 차이)
- **원인**: 정규식이 특정 위치의 키워드만 매칭
- **영향**: 최신성 의도 쿼리에서 recency boost 미적용

### 10.2 [부정확] Half-Life 180일 고정

- **현상**: 빠른 프로젝트(주 단위 변경)에서 30일 전 문서가 "최신" 취급
- **원인**: `HALF_LIFE_MS = 180 * 24 * 60 * 60 * 1000` 하드코딩
- **영향**: 프로젝트 속도와 무관한 최신성 판단

### 10.3 [실패] date 필드 null → 최신성 점수 0

- **현상**: frontmatter에 date 없고 mtime도 null → `getDocRecency()` 반환 0 → recency boost 최저
- **영향**: 새 문서인데 날짜 메타 없으면 가장 오래된 문서 취급

### 10.4 [부정확] mtime 우선 → date 무시

- **현상**: `getDocRecency()`가 mtime 먼저 확인 → 파일 복사/이동으로 mtime이 오늘 → 2021년 문서가 "최신"
- **원인**: mtime 우선 로직
- **영향**: 파일 시스템 조작으로 최신성 왜곡

### 10.5 [부정확] Confluence 배치 동기화 날짜 오염

- **현상**: Confluence 일괄 내보내기 → 모든 문서의 date가 동기화 날짜 → 329개 문서가 "2026-03-11"
- **원인**: refine_html_to_md.py가 modified_date를 우선 사용
- **영향**: BFS 초기 진입점에서 2021년 문서가 "최신"으로 탐색

### 10.6 [부정확] Jira `updated` 필드 오염

- **현상**: 봇 자동 댓글 → 전체 이슈 updated 갱신 → 오래된 이슈가 최근 문서로 인식
- **원인**: Jira updated는 어떤 변경이든 갱신
- **영향**: Jira 동기화 문서의 최신성 신뢰 불가

### 10.7 [부정확] 타임존 불일치

- **현상**: Jira Cloud(UTC) vs 로컬 파일(KST +9시간) → 날짜 비교 시 최대 1일 차이
- **원인**: 타임존 미통일
- **영향**: 증분 동기화에서 경계 시각 이슈 누락

### 14.1 [오류] 검색 설정값 유효성 검사 없음

### 14.4 [부정확] fullVaultThreshold 한국어 미고려

- **현상**: `fullVaultThreshold: 60000` (문자) → 한국어는 영문 대비 정보 밀도 3배 → 실제로는 180K 영문 상당 → RAG가 불필요하게 일찍 트리거
- **영향**: 소규모 한국어 볼트에서 과도한 RAG 오버헤드

### 14.5 [실패] API 키 미설정 시 RAG 실패

- **현상**: LLM API 키 미설정 → `fetchRAGContext()` 내 LLM 호출 실패 → 빈 컨텍스트
- **영향**: 검색 자체는 되지만 LLM 요약/재랭킹 불가

### 15.1 [오류] 멀티 볼트 검색 뮤텍스 데드락

- **현상**: 볼트 A → 볼트 B 전환 → 캐시 미클리어 → 볼트 A 메트릭스가 B 검색에 사용

- **영향**: 잘못된 PageRank 기반 검색

### 15.3 [오류] 동시 fetchRAGContext 호출

- **원인**: `_setDocs(docs)` store mutation 경쟁
- **영향**: 두 검색 모두 비정상 결과

### 15.4 [실패] 볼트 핫 리로드 중 검색

- **현상**: 파일 감시자가 변경 감지 → 볼트 리로드 → 리로드 중 검색 → 부분 로드 상태에서 검색
- **영향**: 일부 문서 누락된 검색 결과

### 15.5 [부정확] _stemCache 메모리 누수

- **원인**: 캐시 프루닝 없음
- **영향**: 장시간 사용 후 앱 속도 저하 → 검색 느려짐

## 16. 그래프 메트릭스

### 16.1 [부정확] PageRank dangling 노드 오분류

- **현상**: 문서 내부 섹션 링크만 있는 문서 → 문서간 외부 링크 없음 → dangling으로 처리
- **원인**: adjacency가 document-level, 내부 섹션 링크 미반영
- **영향**: 실제로 연결된 문서가 dangling으로 PageRank 왜곡

### 16.2 [부정확] 클러스터 감지 — Hub 노드가 클러스터 병합

- **현상**: _index.md 같은 허브가 모든 클러스터를 하나로 병합 → 클러스터 구조 소실
- **원인**: Union-Find가 단순 엣지 기반
- **영향**: "관련 주제 클러스터" 기능 무의미

### 16.3 [부정확] Bridge 노드 정의 불명확

- **현상**: 같은 클러스터에 다중 경로로 연결된 노드 → 이웃 클러스터 1개 → Bridge 아님
- **원인**: `neighborClusters.size >= 2` 조건
- **영향**: 실제 브릿지 노드 누락

### 16.4 [부정확] implicit link 비대칭/비결정적

- **현상**: 문서 1개 추가 → 기존 두 문서간 유사도가 0.25에서 0.24로 하락 → 링크 소실
- **원인**: 코퍼스 변경 시 IDF 재계산으로 유사도 미세 변동
- **영향**: 검색할 때마다 그래프 구조가 미세하게 변경

### 16.5 [오류] 대규모 그래프 메트릭스 계산 시간

- **현상**: 5000+ 노드 → PageRank O(N*E*iter) → 수 초 소요 → UI 프리즈
- **원인**: 메인 스레드에서 동기 계산
- **영향**: 검색 응답 지연
