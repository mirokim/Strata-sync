# Sandbox Map 기능 개선 제안

> 작성일: 2026-03-19
> 현재 버전 기준 분석 (BM25 검색 엔진, Web Worker 인덱싱, Multi-Vault 지원 포함)

---

## 1. 검색 & RAG 품질

### 1-1. BM25 파라미터 UI 노출
**현황**: K1=1.5, B=0.75 하드코딩 (`graphAnalysis.ts`)
**제안**: SearchTab에 K1 / B 슬라이더 추가 (K1: 0.5~3.0, B: 0~1.0)
**효과**: 볼트 성격(전문 용어 밀도, 문서 길이 편차)에 따라 검색 정밀도 조정 가능

### 1-2. 쿼리 자동 확장 (Query Expansion)
**현황**: 입력 쿼리 그대로 검색
**제안**: LLM(Haiku)으로 유사어·영문 약어를 1~3개 보완 후 검색 → 결과 병합
예) "월영 전투 밸런스" → ["월영 combat balance", "전투 시스템 조정"] 추가 검색
**효과**: 볼트 내 영문/혼용 문서 누락 감소

### 1-3. 검색 결과 디버그 패널
**현황**: 왜 이 문서가 나왔는지 볼 수 없음
**제안**: 채팅 응답 하단 "참조 문서 보기" 토글 → 각 문서의 단계별 점수(직접검색/BM25/PPR) 표시
**효과**: 검색 튜닝 시 근거 파악 용이, 사용자 신뢰도 향상

## 2. 그래프 시각화

### 2-1. 태그·폴더 기반 노드 필터링
**현황**: 전체 노드 항상 표시
**제안**: 상단 필터 바에서 태그 / 폴더 / 날짜 범위로 표시 노드 제한
**효과**: 대형 볼트에서 특정 주제 영역만 집중 분석 가능

### 2-2. 클러스터 자동 레이블 + 접기
**현황**: 개별 노드 표시만
**제안**: Union-Find 클러스터(이미 계산 중)에 AI 레이블 생성 + 클러스터 단위로 접기/펼치기
예) "월영 캐릭터 클러스터 (12개)", "전투 시스템 클러스터 (8개)"
**효과**: 수백 노드 볼트에서 구조 파악 시간 단축

### 2-3. 노드 타임라인 모드
**현황**: 공간 배치만
**제안**: X축 = 문서 날짜, Y축 = PageRank 점수로 배치하는 타임라인 뷰 (그래프 모드 토글 추가)
**효과**: 프로젝트 시간 흐름에 따른 지식 진화 파악

### 2-4. 볼트 비교 그래프 개선
**현황**: `CompareGraphCanvas.tsx` 존재하나 기본 구현
**제안**: 두 볼트의 노드 diff 표시 (신규/삭제/변경) + 공통 노드 하이라이트
**효과**: 버전 간 지식 구조 변화 시각적으로 추적

## 3. 채팅 & AI 응답

### 3-1. 채팅 세션 브라우저
**현황**: IndexedDB에 저장되나 UI에서 과거 세션 탐색 불가
**제안**: 왼쪽 패널 또는 Settings에 "과거 대화" 탭 — 날짜별 세션 목록 + 미리보기
**효과**: 이전 분석 결과 재활용, 같은 질문 반복 방지

### 3-2. 응답 내 문서 링크 클릭 시 에디터 이동
**현황**: 참조 문서명이 텍스트로만 표시
**제안**: AI 응답의 `[[문서명]]` 클릭 → 에디터 패널에서 해당 파일 즉시 오픈
**효과**: AI 응답 ↔ 원문 문서 간 이동 흐름 개선

### 3-3. 멀티턴 컨텍스트 핀 고정
**현황**: 대화가 길어지면 앞 내용 자동 요약/압축
**제안**: 특정 메시지에 "핀" 아이콘 → 핀 된 메시지는 컨텍스트 압축 시 항상 유지
**효과**: 중요 지시사항·결정사항이 긴 대화에서도 유지됨

### 3-4. 페르소나별 응답 톤 A/B 비교
**현황**: 선택된 페르소나 한 명이 순차 응답
**제안**: "비교 모드" — 같은 질문을 2~3개 페르소나가 동시 응답, 카드 형태로 나란히 표시
**효과**: 관점 차이를 한눈에 파악

## 4. 에디터

### 4-1. 자동 저장 (Autosave)
**현황**: 수동 저장만
**제안**: 편집 후 2초 디바운스로 자동 저장 + 상태바에 "저장됨 / 저장 중" 표시
**효과**: 작업 중 앱 종료로 인한 데이터 손실 방지

### 4-2. 이미지 드래그앤드롭 삽입
**현황**: 이미지 경로 수동 입력
**제안**: 에디터에 이미지 드래그 → vault 내 assets/ 폴더에 복사 후 `![[image.png]]` 자동 삽입
**효과**: 이미지 관리 워크플로 대폭 단축

### 4-3. WikiLink 유효성 표시
**현황**: 존재하지 않는 WikiLink도 동일하게 표시
**제안**: 볼트에 없는 `[[링크]]`는 빨간 점선 밑줄로 구분 + 호버 시 "파일 없음 — 생성?" 제안
**효과**: 깨진 링크 즉시 인지, 문서 구조 정합성 유지

## 5. Slack 봇

### 5-1. 정기 리포트 스케줄링 UI
**현황**: `scheduledTopics`는 miroStore에 있으나 Slack 정기 전송 UI 없음
**제안**: Settings > Slack 탭에 "매주 월요일 오전 9시 주간 요약 전송" 같은 크론 스케줄 설정 UI
**효과**: 팀 정기 업데이트 자동화

### 5-2. Slack 채널별 페르소나 매핑
**현황**: 페르소나 태그(`[chief]`)로만 구분
**제안**: 채널 ID → 기본 페르소나 매핑 테이블 설정 가능 (예: `#game-art` → art 디렉터 기본)
**효과**: 채널 특성에 맞는 자동 응답 최적화

### 5-3. 웹 검색 결과 출처 신뢰도 표시
**현황**: DuckDuckGo 결과 그대로 포함
**제안**: 검색 결과 도메인 화이트리스트 설정 + 응답에 출처 URL 명시
**효과**: 내부 지식(볼트)과 외부 정보 출처 구분 명확화

---

## 6. MiroFish 시뮬레이터

### 6-1. 시뮬레이션 히스토리 비교
**현황**: 과거 결과 열람만
**제안**: 두 시뮬레이션 결과를 선택해 의견 분포·감성 점수 차이를 시각적으로 비교
**효과**: "조건 A vs 조건 B" 실험 효과 측정 가능

### 6-2. 페르소나 프리셋 가져오기/내보내기
**현황**: 프리셋이 로컬 저장만
**제안**: JSON 파일 내보내기/가져오기 → 팀 간 공유 가능
**효과**: 동일 시뮬레이션 조건 재현성 확보

## 7. 성능 & 안정성

### 7-1. 대형 볼트 가상화 렌더링
**현황**: 파일 트리가 수천 개 항목 전부 DOM에 렌더링
**제안**: 파일 트리에 가상 스크롤(react-virtual 등) 적용
**효과**: 2000개+ 파일 볼트에서 UI 버벅임 제거

### 7-2. 증분 인덱싱
**현황**: 파일 변경 시 전체 BM25 인덱스 재빌드
**제안**: 변경된 파일만 인덱스에서 add/remove/update 처리
**효과**: 대형 볼트에서 파일 저장 후 응답성 향상

### 7-3. 오프라인 동작 보장
**현황**: LLM API 호출 실패 시 에러만 표시
**제안**: API 불가 시 BM25+그래프 검색만으로 "오프라인 RAG 모드" 동작 + UI 배너 표시
**효과**: 네트워크 불안정 환경에서도 로컬 검색 활용 가능
