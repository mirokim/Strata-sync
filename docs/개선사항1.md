
# Code Review: Sandbox-Map 프로젝트
## 프로젝트 개요
**Sandbox-Map**은 Obsidian 마크다운 볼트를 지식 그래프로 시각화하고, 여러 AI 페르소나(디렉터)가 RAG 기반으로 프로젝트 인사이트를 제공하는 하이브리드 데스크톱 애플리케이션입니다.
- **백엔드**: Python FastAPI + ChromaDB (Vector DB)
- **봇**: Python Slack 봇 + OASIS 멀티에이전트 시뮬레이션  
- **프론트**: Electron + Vue 3 + Three.js (3D 그래프 시각화)
- **RAG**: BM25, TF-IDF, PPR(Personalized PageRank), PageRank 조합

## 1. RAG 구현 품질 분석
### 1.1 현황: 잘된 부분
#### BM25 + TF-IDF 조합 (실무적)
- **위치**: `backend/services/chroma_service.py` (ChromaDB + sentence-transformers 활용)
- **장점**:
  - 오프라인 임베딩 모델(all-MiniLM-L6-v2, ~90MB) 사용 → API 비용/지연 없음

- **위치**: `bot/modules/rag_simple.py` lines 57-72
- **구현**: `sigmoid(log1p(count)) × exp(-decay × age_days)`
- **강점**:
  - OpenViking 이론 기반 → 학술적 기초 있음
  - 접근 횟수 + 최신성을 동시에 반영
  - Half-life (7일) 파라미터로 감쇠율 커스터마이징 가능
  - Lock 기반 동시성 제어 (concurrent read-modify-write 보호)

### 1.2 현황: 개선 필요 부분
#### HIGH: TF-IDF 재구현의 단순성 문제
**파일**: `bot/modules/rag_simple.py` lines 97-108 (`_score_doc`)
```python
def _score_doc(doc: VaultDoc, query_tokens: list[str]) -> float:
    title_lower = doc.title.lower()
    stem_lower  = doc.stem.lower()
    body_lower  = doc.body.lower()
    score = 0.0
    for token in query_tokens:
        if token in title_lower: score += 3.0
        if token in stem_lower:  score += 2.0
        count = body_lower.count(token)
        if count > 0:
            score += min(count * 0.5, 3.0)
    return score
```
**문제점**:
1. **TF-IDF 명칭과 불일치**: "TF-IDF"라고 자주 언급하지만 실제는 **단순 가중 TF (Term Frequency)**만 구현
   - IDF(Inverse Document Frequency) 없음 → 공통 단어("프로젝트", "문서" 등) 과가중
   - 예: "프로젝트"를 찾으면 모든 문서가 높은 점수 → 구분력 없음
2. **토큰화 문제**: 형태소 분석 미흡
   - 정규식 기반 단순 분할: `re.split(r"[\s\[\](),./|_\-]+", text.lower())`
   - 한글 "~하기", "~해주는" 등 문법형태 미분리 → 검색 누락 가능
   - 예: "개선하는" vs "개선" 다른 토큰으로 취급
3. **문자 위치 기반 가중치 간단함**:
   - Title(3.0) > Stem(2.0) > Body(0.5 per count) → 하드코딩
   - 동적 조정 메커니즘 없음
**개선 방향**:

```python
import time
from typing import Optional
def _analyze_doc_with_retry(
    client, query: str, doc: dict, idx: int,
    max_retries: int = 2,
    log_fn=None,
) -> dict:
    """재시도 로직 포함 서브에이전트 분석"""
    for attempt in range(max_retries):
        try:
            raw = client.complete(_SUB_AGENT_SYSTEM, user_prompt, max_tokens=400)
            data = json.loads(_extract_json(raw))
            return {
                "idx": idx,
                "score": float(data.get("score", 0)),
                "summary": data.get("summary", ""),
                "key_points": data.get("key_points", []),
            }
        except json.JSONDecodeError as e:
            if log_fn:
                log_fn(f"[서브에이전트 #{idx}] JSON 파싱 실패 (시도 {attempt+1}): {e}")
            if attempt == max_retries - 1:
                break
            time.sleep(1)  # exponential backoff
        except (ConnectionError, TimeoutError) as e:
            if log_fn:
                log_fn(f"[서브에이전트 #{idx}] 네트워크 오류 (시도 {attempt+1}): {e}")
            if attempt < max_retries - 1:
                time.sleep(2 ** attempt)  # exponential backoff
    
    # 폴백
    return _fallback_analysis(doc, idx)
```

**문제점**:
1. **30분 TTL**: 장시간 시뮬레이션(예: 100명 × 20라운드) 중단 후 30분 후 재개 시 모두 재분석

2. **재계산 감지 없음**: Query/stem 세트가 동일해도 TTL 만료 시 모두 버림
**개선 제안**:
```python
_CHECKPOINT_TTL_SECS = 86400 * 7  # 7일로 연장
# 또는 용량 기반 LRU 캐시
from functools import lru_cache
import json
class CheckpointStore:
    MAX_SIZE_MB = 100
    
    def save_checkpoint(self, key: str, analyses: list[dict]) -> None:
        os.makedirs(_CHECKPOINT_DIR, exist_ok=True)
        path = os.path.join(_CHECKPOINT_DIR, f"{key}.json")
        slim = [{"idx": a["idx"], "score": a["score"], ...} for a in analyses]
        
        with open(path, "w") as f:
            json.dump(slim, f)
        
        # 디렉토리 크기 초과 시 오래된 파일부터 삭제 (LRU)
        self._cleanup_if_needed()
    
    def _cleanup_if_needed(self) -> None:
        total = sum(f.stat().st_size for f in Path(_CHECKPOINT_DIR).glob("*.json"))
        if total > self.MAX_SIZE_MB * 1024 * 1024:
            # 오래된 순서대로 삭제
            files = sorted(Path(_CHECKPOINT_DIR).glob("*.json"), 
                          key=lambda f: f.stat().st_mtime)
            for f in files:
                if total <= self.MAX_SIZE_MB * 1024 * 1024 * 0.8:
                    break
                f.unlink()
                total -= f.stat().st_size
```
