# RAG 데이터 정제 가이드

> 작성일: 2026-03-13
> 대상: Sandbox Map 볼트 데이터 작업자
> 목적: Slack 봇 / 앱 내 AI 검색이 최신 문서를 정확히 찾아올 수 있도록 데이터 구조 요건 정리

---

## 1. RAG 검색 동작 방식 (기술 요약)

질문이 들어오면 두 단계로 문서를 찾습니다.

### Stage 1 — 키워드 직접 검색 (`directVaultSearch`)

쿼리 단어를 **파일명**과 **본문**에서 찾아 점수를 매깁니다.

| 매칭 위치 | 가중치 |
|----------|--------|
| 파일명에 포함 | +2점 |
| 본문에 포함 | +1점 |

예) 쿼리 "캐릭터C 최신 자료 2026 03 13"

| 문서 | 파일명 점수 | 본문 점수 | 합계 |
|------|------------|----------|------|
| `ProjectA_정례_20240215_캐릭터C맵_일월봉.md` | "캐릭터C" +2 = 2 | "캐릭터C" +1 = 1 | 3 |
| `[2026.01.15]_캐릭터C_3차_정례.md` | "2026" +2, "캐릭터C" +2 = 4 | "캐릭터C" +1 = 1 | 5 ✓ |

→ **파일명에 연도(2026, 2025 등)가 있으면 최신 쿼리에서 자동 우선순위 상승**

### Recency Boost (날짜 기반 가중치)

"최신/최근/요즘/이번달/지금/현재" 등의 인텐트가 감지되면:

```
최종점수 = 키워드점수 + 2.0 × e^(-(오늘 - 문서날짜) / 180일)
```

| 문서 날짜 | boost 값 |
|----------|---------|
| 2026-01-15 (58일 전) | ≈ 1.52 |
| 2025-09-01 (193일 전) | ≈ 0.52 |
| 2024-02-15 (757일 전) | ≈ 0.03 |

→ **최신 문서가 boost로 역전 가능. 단, frontmatter `date` 필드가 없으면 boost가 작동하지 않음.**

> ⚠️ **핵심**: `mtime`(파일 수정 시간)을 우선 사용하지 않고 frontmatter `date`를 우선 사용합니다.
> Confluence 동기화 시 모든 파일의 mtime이 오늘로 갱신되기 때문에 mtime으로는 2024/2026 문서를 구분할 수 없습니다.

### Stage 2 — BFS 그래프 탐색

Stage 1에서 찾은 문서를 시드(seed)로, 위키링크(`[[문서명]]`)를 따라 연결된 문서를 최대 3홉까지 수집합니다. "최신" 인텐트 시 시드를 날짜 역순 정렬 후 BFS합니다.

## 2. 발견된 문제 (현재 버그)

### 문제 증상

- "캐릭터C 최신 내용 알려줘" → **2024년 문서** 반환
- "캐릭터C 2026년 자료 알려줘" → **2026년 문서** 정상 반환

### 근본 원인 분석

**원인 1: `date` frontmatter 누락 또는 오류**

```yaml
# ❌ 잘못된 예 — date 없음
---
title: "캐릭터C 3차 정례회의"
type: meeting
tags: ["캐릭터C", "정례"]
---
```

```yaml
# ✅ 올바른 예
---
title: "캐릭터C 3차 정례회의"
date: 2026-01-15
type: meeting
tags: ["캐릭터C", "정례"]
---
```

date가 없으면 recency boost = 0 → 오래된 문서와 구분 불가.

---

**원인 2: 파일명에 연도 없음**

"캐릭터C 2026년" 처럼 연도를 명시한 쿼리만 2026 문서를 찾음.
"캐릭터C 최신" 처럼 연도 없는 쿼리는 키워드 점수만으로 판단 → 본문에 "캐릭터C" 언급이 많은 오래된 문서가 상위권.

```
# ❌ 연도 정보가 없는 파일명
캐릭터C_3차_정례회의록.md

# ✅ 연도가 포함된 파일명 (검색 점수 +2)
[2026.01.15]_캐릭터C_3차_정례.md
20260115_캐릭터C_3차_정례회의록.md
캐릭터C_정례_2026_01_15.md
```

---

**원인 3: `type` 필드 오류 또는 누락**

BFS 탐색 시 문서 유형(meeting, reference, report 등)이 컨텍스트 헤더에 표시됩니다. 잘못 분류된 경우 LLM이 관련성을 낮게 판단할 수 있습니다.

---

**원인 4: 위키링크 미연결**

2026년 캐릭터C 문서들이 서로 연결되지 않으면 BFS가 관련 문서를 수집하지 못합니다.

```markdown
# ❌ 독립 문서
이번 회의에서 논의한 사항...

# ✅ 연결된 문서
이번 회의에서 논의한 사항 ([[캐릭터C_2차_정례회의록]] 참고)...
```

## 3. 데이터 정제 체크리스트

### 3-1. Frontmatter 필수 필드

모든 볼트 문서는 다음 frontmatter를 가져야 합니다:

```yaml
---
title: "문서 제목"           # 필수 — 검색 결과 표시명
date: YYYY-MM-DD            # 필수 — recency boost 기준 (실제 회의/작성일)
type: meeting               # 필수 — meeting / report / reference / decision / plan
status: active              # 권장 — active / archived
tags: ["프로젝트명", "유형"] # 권장 — 검색 정확도 향상
---
```

**`type` 값 가이드:**

| type | 사용 케이스 |
|------|-----------|
| `meeting` | 회의록, 정례회의, 킥오프 |
| `report` | 보고서, 분석, 결과물 |
| `reference` | 참고자료, 가이드, 스펙 |
| `decision` | 의사결정 문서, 승인 내용 |
| `plan` | 계획서, 로드맵 |

---

### 3-2. 파일명 규칙

**권장 형식:**

```
[YYYY.MM.DD]_프로젝트명_문서유형.md
YYYYMMDD_프로젝트명_문서유형.md
```

**예시:**

```
[2026.01.15]_캐릭터C_3차정례.md
[2026.02.10]_캐릭터C_퀘스트설계_검토.md
[2026.03.05]_캐릭터C_환경아트_현황.md
20260115_캐릭터C_킥오프.md
```

**파일명에 반드시 포함할 것:**
- 연도 (2026, 2025 등) — 최신 쿼리 키워드 매칭의 핵심
- 프로젝트/지역명 (캐릭터C, 일월봉 등)
- 문서 유형 (정례, 보고, 설계 등)

---

### 3-3. 위키링크 연결 구조

같은 프로젝트 문서들은 서로 연결해야 BFS가 관련 문서를 따라가며 수집할 수 있습니다.

**권장 패턴:**

```markdown
## 이전 회의 참고
- [[2026.01.15_캐릭터C_2차정례]] — 환경아트 방향 결정
- [[2026.02.10_캐릭터C_퀘스트설계_검토]] — 퀘스트 목록 확정

## 후속 액션
- [[캐릭터C_환경아트_현황]] 문서 업데이트 예정
```

**`_index.md` 활용:**

볼트에 `_index.md` 또는 프로젝트별 `index_캐릭터C.md`가 있으면 BFS 시작점으로 자주 활용됩니다. 최신 문서들로의 링크를 이 파일에 관리하세요.

---

### 3-4. 날짜 형식 통일

frontmatter `date` 필드는 **ISO 8601 형식**만 사용합니다:

```yaml
date: 2026-01-15    # ✅ 올바른 형식

date: 2026/01/15    # ❌ 슬래시 → 파싱 실패 가능
date: 26-01-15      # ❌ 2자리 연도 → 파싱 오류
date: 2026년 1월 15일  # ❌ 한국어 → 파싱 실패
date: "Jan 15, 2026"   # ❌ 영문 → 파싱 실패
```

## 4. Confluence 동기화 주의사항

Confluence에서 가져온 문서는 자동으로 frontmatter가 생성됩니다:

```yaml
date: {Confluence 페이지 최종 수정일}   # version.when 기준
```

**문제가 생기는 경우:**

1. **Confluence 페이지에 수정이 없었던 경우** — 생성일(createdDate)이 date로 들어옴. 내용은 2026년이지만 페이지 생성이 2024년이면 date = 2024
2. **일괄 페이지 이동/복사** — 모든 페이지의 version.when이 이동일로 업데이트될 수 있음
3. **자동화 스크립트로 생성된 페이지** — 스크립트 실행일이 date로 들어옴

**해결:** Confluence 페이지의 `version.when`을 실제 회의/작성일로 맞추거나, 동기화 후 볼트 파일의 `date` 필드를 수동으로 수정.

## 5. 빠른 점검 방법

### 특정 프로젝트 문서 date 일괄 확인

볼트 루트에서:

```bash
# 캐릭터C 관련 문서의 date 필드 추출
grep -r "^date:" /path/to/vault --include="*.md" | grep -i "캐릭터C"

# date 없는 파일 찾기
grep -rL "^date:" /path/to/vault --include="*.md"
```

### RAG가 어떤 문서를 찾았는지 확인

Sandbox Map 앱 → 설정 → AI 설정에서 **디버그 로그**를 켜면 Stage 1/Stage 2에서 선택된 문서 목록이 콘솔에 출력됩니다.

---

## 6. 권장 정제 작업 순서

1. **date 필드 점검** — `grep -rL "^date:"` 로 누락 파일 찾아 추가
2. **date 값 검증** — 실제 작성/회의일과 일치하는지 확인 (Confluence 이동/복사 이력 주의)
3. **파일명 연도 추가** — 연도 없는 파일명에 연도 접두사 추가
4. **위키링크 연결** — 같은 프로젝트 문서끼리 `[[링크]]` 연결
5. **`_index.md` 정비** — 최신 문서들로의 링크 최신화
6. **동기화 후 재테스트** — Slack에서 "프로젝트명 최신 자료 알려줘" 쿼리로 검증

---

## 7. 요약: 핵심 규칙 3가지

> **① frontmatter `date`는 반드시 실제 작성일 (YYYY-MM-DD)**
> → 없으면 최신 쿼리에서 오래된 문서와 구분 불가
>
> **② 파일명에 연도 포함**
> → "캐릭터C 2026" 없이 "캐릭터C 최신"으로도 찾히려면 파일명에 연도 있어야 함
>
> **③ 같은 프로젝트 문서는 위키링크로 연결**
> → BFS가 연결을 따라가며 관련 문서를 모아줌
